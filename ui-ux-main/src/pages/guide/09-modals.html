Modals

Struktur modal untuk tambah data, ubah data, konfirmasi hapus, detail singkat, dan review data.

Ringkasan Ukuran Modal Anatomi Contoh Modal Panduan Pakai Implementasi

Ringkasan

Modal memakai backdrop, panel utama, header, body, dan footer action di bagian bawah. Default aplikasi memakai ukuran md, sedangkan modal permission atau matrix besar memakai lg.

Centered · Footer radius · Accessible trigger

Centered Modal

Modal tampil di tengah layar, bukan drawer, untuk form tambah/ubah master data dan detail ringkas.

Sticky Footer

Action utama selalu ada di footer modal memakai modal-actions.

Confirm Modal

Untuk aksi destructive seperti hapus data, tampilkan modal konfirmasi lebih dulu.

Ukuran Modal

Gunakan class ukuran pada elemen .modal-card. Tanpa modifier ukuran, modal mengikuti ukuran standar md.

Ukuran Class Lebar Rekomendasi Penggunaan
SM .is-sm 420px Konfirmasi hapus, warning kecil, validasi singkat, resend OTP.
MD .is-md / default 760px Tambah/ubah master data standar seperti unit, kamar, tarif, karyawan, dan penjamin.
LG .is-lg 980px Form 2 kolom yang lebih padat, detail panjang, role permission, atau data dengan table kecil.
XL .is-xl 1120px Matrix lebar, preview laporan, atau konten kompleks yang memang butuh ruang ekstra.

Anatomi Modal

Struktur modal harus konsisten: backdrop, wrapper modal, panel, header, body, dan footer action.

Struktur HTML

<div class="modal-backdrop"></div>
<section class="modal">
  <div class="modal-card is-md">
    <div class="modal-head">...</div>
    <div class="modal-body">...</div>
    <div class="modal-actions">...</div>
  </div>
</section>

Tambah Unit

Header berisi title dan deskripsi singkat.

Footer action tetap di bawah saat body modal scroll.

Contoh Modal

Pilih contoh berikut untuk melihat pola title, deskripsi, content, dan tombol action.

Panduan Pakai

Aturan praktis supaya modal tidak terlalu ramai dan tetap konsisten di halaman admin/master data.

Gunakan modal-actions untuk footer.

Tombol Batal/Simpan tetap rata kanan dengan jarak yang sama.

Pilih ukuran sesuai konten.

Form master data cukup MD, matrix permission boleh LG, konfirmasi cukup SM.

Isi title dan subtitle dengan konteks jelas.

User harus langsung tahu data apa yang sedang dibuat, diubah, atau dihapus.

Jangan langsung menjalankan delete.

Aksi hapus wajib lewat confirm modal dengan copy yang jelas dan tombol destructive.

Jangan memakai modal XL untuk form pendek.

Ukuran terlalu besar membuat fokus user melebar tanpa kebutuhan.

Jangan menaruh scroll di seluruh page.

Konten panjang harus scroll di modal-body, bukan membuat modal keluar viewport.

Implementasi CSS, Tailwind, dan React

Modal memakai class modal-* sebagai struktur utama dialog, mulai dari backdrop, panel, body, sampai footer action.

design-system/modals.css

CSS / Tailwind v4 source

/* src/css/design-system/modals.css */
.modal-backdrop { @apply fixed inset-0 z-[80] bg-[color:rgba(15,31,61,0.46)]; backdrop-filter: blur(4px); }
.modal-backdrop[hidden] { @apply hidden; }
.modal { @apply fixed inset-0 z-[90] grid place-items-center p-6; }
.modal[hidden] { @apply hidden; }
.modal-card { @apply w-[min(760px,calc(100vw_-_48px))] min-w-0 p-0 border-0 rounded-[var(--radius-card)] bg-[color:var(--surface)] shadow-[0_30px_90px_rgba(15,31,61,0.24)]; }
.modal-card.is-sm { @apply w-[min(420px,calc(100vw_-_48px))]; }
.modal-card.is-lg { @apply w-[min(980px,calc(100vw_-_48px))]; }
.modal-head { @apply flex items-start justify-between gap-3 min-w-0 pt-6 pb-3.5 px-6 m-0 border-b border-b-[color:var(--border-soft)]; }
.modal-body { @apply grid gap-4 min-w-0 pt-5 pb-6 px-6; }
.modal-actions { @apply flex items-center justify-end gap-3 min-w-0 m-0 pt-3.5 pb-5 px-6 bg-[color:var(--surface)] border-t border-t-[color:var(--border-soft)] rounded-[0_0_var(--radius-card)_var(--radius-card)]; }

HTML / React mapping

<div class="modal-backdrop" hidden></div>
<section class="modal" role="dialog" aria-modal="true" aria-labelledby="modalTitle" hidden>
  <article class="modal-card is-md">
    <header class="modal-head">
      <h2 id="modalTitle" class="modal-title">Tambah Unit</h2>
      <button class="btn btn-ghost btn-icon" type="button" aria-label="Tutup">...</button>
    </header>
    <div class="modal-body">...</div>
    <footer class="modal-actions">
      <button class="btn btn-secondary">Batal</button>
      <button class="btn btn-primary">Simpan</button>
    </footer>
  </article>
</section>

function Modal({ open, title, size = 'is-md', children, footer }) {
  return open ? <section className="modal" role="dialog" aria-modal="true">...</section> : null
}
Catatan: Di React, state modal cukup mengatur buka/tutup. Class visual tetap memakai struktur yang sama.
