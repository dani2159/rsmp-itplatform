Anatomi Komponen

Struktur dasar sidebar yang perlu konsisten di semua dashboard SIMRS.

01

Header sidebar

Berisi logo/brand, nama produk atau RS, dan aksi switch/collapse memakai icon button.

02

Item navigasi

Setiap item terdiri dari icon, label, badge count opsional, chevron opsional, dan state active/hover/focus.

03

Mobile drawer

Saat responsive, sidebar berubah menjadi drawer dari kiri dengan overlay. Logo tetap di header, bukan di dalam drawer.

Aturan Penggunaan

Aturan penggunaan agar sidebar tetap konsisten dan tidak membuat layout terasa penuh.

Yang disarankan

  • Gunakan active state hanya untuk satu parent atau satu leaf page.
  • Gunakan badge count untuk item yang benar-benar perlu perhatian pengguna.
  • Gunakan collapsed sidebar saat ruang horizontal terbatas, tetap sediakan tooltip.
  • Gunakan FontAwesome: sidebar memakai fa-solid / filled, sedangkan top navbar memakai gaya line/outline seperti fa-regular bila tersedia.

Yang dihindari

  • Jangan memakai lebih dari dua level nested navigation di sidebar.
  • Jangan campur style sidebar gelap dan terang dalam satu aplikasi.
  • Jangan gunakan warna status merah untuk menu inactive kecuali ada error.
  • Jangan hilangkan focus state untuk navigasi keyboard.

Implementasi CSS, Tailwind, dan React

Sidebar memakai .app-sidebar, .sidebar-user-info, .nav-item, .nav-group, dan .subnav. Top nav memakai .topnav-shell, .topnav-utilitybar, .topnav-main, .topnav-link, dan .topnav-dropdown.
