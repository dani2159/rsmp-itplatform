Ringkasan

Button harus membantu pengguna mengambil keputusan dengan cepat. Gunakan hierarki visual yang konsisten: satu aksi utama, aksi sekunder secukupnya, dan aksi destruktif harus terlihat jelas.

Grid 4px · radius 8px · focus terlihat
Primary hanya untuk aksi utama.
Contoh: Simpan, Daftarkan Pasien, Buat Resep, Bayar Tagihan.
Ukuran large tidak dipakai di form mobile.
Gunakan MD sebagai default. LG/XL hanya untuk desktop CTA, kiosk, atau display khusus.
Icon-only wajib memakai aria-label.
Icon tanpa teks harus punya label aksesibilitas dan tooltip untuk menghindari salah klik.

Icon, Icon-only, dan Loading

Gunakan icon untuk mempercepat pemahaman, bukan dekorasi. Icon-only hanya untuk aksi yang sudah umum atau berada di konteks tabel yang jelas.

Anatomi

Icon Pattern

Icon kiri untuk create/add.
Icon kanan untuk lanjut/continue.
Icon-only wajib aria-label.
Saat loading, klik dinonaktifkan sampai proses selesai.

Panduan Penggunaan

Aturan ini menjaga konsistensi di semua modul SIMRS, terutama pada form medis, tabel pasien, modal konfirmasi, dan alur transaksi.

Yang disarankan

Gunakan satu primary per section.

Primary menjadi fokus utama, secondary/outline sebagai pendukung.

Label harus berupa aksi.

Pakai “Simpan”, “Daftarkan”, “Export”, bukan label ambigu seperti “OK”.

Loading mengganti label secara jelas.

Contoh: “Menyimpan”, “Menghapus”, “Memproses Pembayaran”.

Danger selalu butuh konfirmasi.

Untuk hapus/batal/tolak, tampilkan modal konfirmasi sebelum eksekusi.

Yang dihindari

Jangan pakai primary terlalu banyak.

Lebih dari satu primary di area yang sama membuat pengguna bingung.

Jangan pakai warna custom.

Semua warna harus berasal dari token brand atau semantic.

Jangan buat icon-only tanpa label.

Wajib aria-label dan sebaiknya ada tooltip.

Jangan pakai LG di mobile form.

Gunakan MD atau full-width MD untuk mobile, bukan LG/XL.

Implementasi CSS, Tailwind, dan React

Button memakai .btn sebagai base, lalu ditambah varian dan ukuran. Style komponen dikelola di buttons.css melalui @apply.

const variantClass = {
  primary: 'btn-primary',
  secondary: 'btn-secondary',
  outline: 'btn-outline',
  ghost: 'btn-ghost',
  danger: 'btn-danger',
  success: 'btn-success',
  warning: 'btn-warning',
}
const sizeClass = { xs: 'btn-xs', sm: 'btn-sm', md: 'btn-md', lg: 'btn-lg', xl: 'btn-xl' }

function Button({ variant = 'primary', size = 'md', iconOnly, loading, className = '', ...props }) {
  const classes = ['btn', variantClass[variant], sizeClass[size], iconOnly && 'btn-icon', loading && 'is-loading', className]
    .filter(Boolean)
    .join(' ')
  return <button className={classes} {...props} />
}
Catatan: Class HTML lama seperti btn btn-primary btn-md tetap dipertahankan. Tailwind utility masuk di CSS source memakai @apply.
