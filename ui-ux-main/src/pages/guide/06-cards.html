Panduan Pakai

Aturan sederhana supaya card konsisten di semua halaman dashboard dan form operasional.

Pakai .card sebagai base component.

Card utama, statistik, dan form wrapper tetap berangkat dari class yang sama.

Gunakan .card-danger hanya untuk status kritis.

Cocok untuk JKN tertolak, tunggakan, error, atau kondisi yang perlu perhatian cepat.

Letakkan action di .card-footer.

Footer menjaga jarak tombol dan link tetap konsisten di semua card.

Jangan buat wrapper card baru jika .card cukup.

Modifier dan utility lebih aman daripada menambah pola card lain.

Jangan memakai colored card hanya untuk dekorasi.

Warna card harus punya fungsi status atau penekanan informasi.

Jangan menaruh terlalu banyak action di header.

Header cukup untuk title, subtitle, badge, atau satu aksi ringan.

Implementasi CSS, Tailwind, dan React

Card memakai .card sebagai kontainer utama, dengan .card-header, .card-title, .card-text, dan .card-footer untuk struktur isi.

cards.css

CSS / Tailwind v4 source

/* src/css/design-system/cards.css */
.card {
  @apply border-0 bg-[color:var(--surface)] rounded-[var(--radius-card)] p-4 shadow-[var(--shadow-sm)] min-w-0;
}

.card-header,
.card-footer {
  @apply flex items-start justify-between gap-3 min-w-0;
}

.card-title {
  @apply m-0 text-[color:var(--text-primary)] text-body-lg leading-6 font-bold;
}

.card-text,
.card p.card-text {
  @apply m-0 text-[color:var(--text-secondary)] text-body-md leading-5.5 font-normal;
}

.card-tone-icon {
  @apply w-8 h-8 rounded-xl inline-flex items-center justify-center flex-none bg-[color:var(--primary-tint)] text-[color:var(--primary-blue)];
}

HTML / React mapping

<article class="card">
  <div class="card-header">
    <div>
      <h3 class="card-title">Ringkasan Booking</h3>
      <p class="card-text">Bed, penjamin, dan status pasien.</p>
    </div>
    <span class="badge badge-success badge-sm">Aktif</span>
  </div>
  <div class="card-footer">
    <a class="card-link" href="#">Lihat detail</a>
  </div>
</article>

function Card({ title, subtitle, badge, children, footer }) {
  return (
    <article className="card">
      <div className="card-header">
        <div>
          <h3 className="card-title">{title}</h3>
          {subtitle && <p className="card-text">{subtitle}</p>}
        </div>
        {badge}
      </div>
      {children}
      {footer && <div className="card-footer">{footer}</div>}
    </article>
  )
}
Catatan: Untuk card halaman spesifik boleh tambah wrapper page-level, tetapi struktur card dasar tetap pakai class ini.
