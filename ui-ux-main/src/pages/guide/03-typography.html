Panduan Penggunaan

Aturan praktis untuk menjaga konsistensi halaman dengan data padat.

Yang disarankan & dihindari
01

Jaga hierarki halaman

  • Gunakan satu H1 per halaman.
  • Gunakan H2 untuk section utama.
  • Gunakan H3/H4 untuk card dan group kecil.
02

Body default 14px

  • Gunakan 14px untuk mayoritas UI.
  • Gunakan line-height 22px agar teks tidak padat.
  • Hindari body 12px untuk paragraf panjang.
03

Monospace hanya untuk kode

  • Gunakan untuk RM, ICD, SEP, hasil lab.
  • Jangan gunakan untuk paragraf umum.
  • Pakai semibold/bold untuk ID penting.
04

Batasi weight

  • 400 untuk body.
  • 500 untuk value/menu.
  • 600 untuk heading kecil dan label.
  • 700 untuk H1 dan emphasis penting.
05

Jangan terlalu banyak uppercase

  • Uppercase hanya untuk label pendek.
  • Tambahkan tracking .05em untuk label.
  • Jangan uppercase untuk kalimat panjang.
06

Readable di tabel padat

  • Header tabel 12px semibold/bold.
  • Cell default 14px regular/medium.
  • ID teknis pakai mono-sm atau mono-md.

Implementasi CSS, Tailwind, dan React

Tipografi memakai Inter untuk interface dan JetBrains Mono untuk data teknis. Class seperti .text-h1, .text-body-xl, dan .text-body-md menjadi acuan ukuran teks.

typography.css

CSS / Tailwind v4 source

/* src/css/design-system/theme.css */
@theme {
  --font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
  --font-mono: "JetBrains Mono", ui-monospace, monospace;
  --text-h1: 32px;
  --text-h1--line-height: 40px;
  --text-body-md: 14px;
  --text-body-md--line-height: 22px;
}

/* src/css/design-system/typography.css */
.text-h1 {
  @apply text-h1 leading-[var(--leading-h1)] tracking-[var(--tracking-h1)] font-bold;
}

.text-body-md {
  @apply text-body-md leading-[var(--leading-body-md)];
}

.text-mono-sm {
  @apply font-mono text-mono-sm leading-[var(--leading-mono-sm)] tracking-[var(--tracking-mono-sm)];
}
